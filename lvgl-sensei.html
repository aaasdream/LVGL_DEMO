LVGL 是什麼東西？
先搞清楚你在用啥，再說其他的。
👩‍💻
LVGL = Light and Versatile Graphics Library。
翻成人話：一個專門為嵌入式系統（單晶片、MCU）設計的圖形介面函式庫。

你平常用的 React、Qt 那些？用不了，記憶體根本不夠。LVGL 只要 64KB RAM 和 256KB Flash 就能跑， 這才是在 Arduino / ESP32 上做漂亮 UI 的正確姿勢。

完全免費、MIT 授權、C 語言寫的——連你這種 C++ 半吊子也能用。
🏗️

Widget 系統

按鈕、滑桿、圖表、選單…幾十種現成元件，不用自己手刻像素。

🎨

Style 樣式

類似 CSS 的概念，一個 Style 可以套到多個物件，改起來不要太方便。

⚡

動畫引擎

內建 easing 函式、動畫佇列，嵌入式裡做到 60fps 滑順動畫不是夢。

👆

事件驅動

觸控、按鍵、滑鼠全部統一事件介面，callback 寫起來乾乾淨淨。

🔌

驅動無關

只要你實作 flush callback，管你用什麼螢幕 IC，SPI、I2C 都行。

🌐

Arduino 支援

官方有 Arduino library，ESP32 / STM32 社群支援超完整。

👩‍💻
你要知道 LVGL 分成三層：
你寫的 App（Widget 操作） → LVGL 核心（渲染、事件） → 硬體驅動（你要自己接）。

最下面那層驅動，LVGL 完全不管，它只呼叫你提供的 callback， 所以換螢幕不用改 LVGL，只改你的驅動函式就好。聰明設計吧？

核心概念：物件、樣式、事件
三個東西搞懂，你就能做出 90% 的 UI。
👩‍💻
LVGL 的一切都是「物件（Object / lv_obj_t）」。
你創一個按鈕，它是物件。在按鈕上放文字，也是物件。螢幕本身？也是物件。

每個物件都有父子關係——子物件的座標是相對於父物件的， 父物件移動、子物件跟著動。裁剪範圍也自動限制在父範圍內。

說白了就是一棵樹，根節點是 lv_screen_active()，往下長就對了。

Style（樣式） 的概念你可以想成 CSS。
背景色、邊框、圓角、內距、陰影⋯全部叫做 style property。

重點在 State（狀態）：同一個按鈕， LV_STATE_DEFAULT 是普通樣式， LV_STATE_PRESSED 是按下去的樣式， LV_STATE_CHECKED 是選中樣式—— LVGL 自動在不同狀態切換，你只要設定好就行。
🎨

樣式屬性（Style Properties）

背景色、邊框、圓角、文字色、字型、Padding、陰影、透明度⋯全部可以控制。使用 lv_obj_set_style_xxx() 系列函式。

🔀

Part（部件）

複雜 Widget 有多個 Part：例如 Slider 有 MAIN（底軌）、INDICATOR（進度）、KNOB（旋鈕），可以分別設定樣式。

📡

事件（Event）

按下、放開、值變更、長按⋯用 lv_obj_add_event_cb() 綁定 callback，邏輯清楚不雜亂。

event_example.c
// ── 建立按鈕並綁定事件（學姊說這是基本功）──

static void btn_cb(lv_event_t *e) {
    lv_event_code_t code = lv_event_get_code(e);
    lv_obj_t *btn = lv_event_get_target_obj(e);

    if (code == LV_EVENT_CLICKED) {
        // 取得按鈕上的標籤
        lv_obj_t *label = lv_obj_get_child(btn, 0);
        
        // 切換文字（簡單狀態機）
        static bool on = false;
        on = !on;
        lv_label_set_text(label, on ? "ON ✓" : "OFF");
        
        // 切換背景色（直接在物件上設 inline style）
        lv_obj_set_style_bg_color(btn,
            on ? lv_color_hex(0x4ade80) : lv_color_hex(0x334155),
            LV_PART_MAIN | LV_STATE_DEFAULT);
    }
}

void my_ui(void) {
    lv_obj_t *scr = lv_screen_active();

    // 建立按鈕
    lv_obj_t *btn = lv_button_create(scr);
    lv_obj_set_size(btn, 120, 50);
    lv_obj_center(btn);

    // 圓角樣式
    lv_obj_set_style_radius(btn, 25, LV_PART_MAIN | LV_STATE_DEFAULT);
    lv_obj_set_style_bg_color(btn, lv_color_hex(0x334155),
                              LV_PART_MAIN | LV_STATE_DEFAULT);
    // 按下去變暗
    lv_obj_set_style_bg_color(btn, lv_color_hex(0x1e293b),
                              LV_PART_MAIN | LV_STATE_PRESSED);

    // 標籤
    lv_obj_t *lbl = lv_label_create(btn);
    lv_label_set_text(lbl, "OFF");
    lv_obj_center(lbl);

    // 綁事件
    lv_obj_add_event_cb(btn, btn_cb, LV_EVENT_CLICKED, NULL);
}

函式庫安裝三步驟，不要搞錯順序：
1

Arduino IDE → 函式庫管理 → 搜尋 TFT_eSPI by Bodmer → 安裝

⚠️ 重要 安裝後要去 Arduino/libraries/TFT_eSPI/User_Setup.h 設定你的螢幕 IC 型號、SPI 腳位！預設值幾乎都是錯的。
2

安裝 LVGL

函式庫管理 → 搜尋 lvgl → 安裝（目前推薦 v9.x）

ℹ️ 必做步驟 安裝後複製一份設定檔：把 libraries/lvgl/lv_conf_template.h 複製到 libraries/lv_conf.h（注意位置）， 然後把第一行 #if 0 改成 #if 1。
3

確認 lv_conf.h 關鍵設定

最重要的事：接好驅動
LVGL 不管你用什麼螢幕，它只要你提供這兩個 callback，你接好了它才動得起來。
👩‍💻
LVGL 的設計哲學是：「我只負責算出要畫什麼像素，傳出去是你家的事」。

所以你要提供：
① flush_cb：LVGL 算好了一塊畫面，你把它塞進 SPI 傳給螢幕。
② touchpad_read_cb：LVGL 問你「現在有沒有人按著？座標哪裡？」你回答它。

就這樣。兩個函式搞定所有硬體橋接，多優雅。

lvgl_arduino_template.ino — 完整模板（學姊親手寫的，珍惜）
/**
 * LVGL + Arduino / ESP32 標準模板
 * 硬體：ESP32 + ILI9341 (320×240) + XPT2046 觸控
 * 函式庫：TFT_eSPI + lvgl 9.x
 */

#include <lvgl.h>
#include <TFT_eSPI.h>

// ────────────────────────────────────
//  硬體物件
// ────────────────────────────────────
TFT_eSPI tft = TFT_eSPI();

// ── 觸控 SPI 腳位（XPT2046）──
#define TOUCH_CS   33
#define TOUCH_IRQ  36

// ────────────────────────────────────
//  LVGL 顯示 Buffer（建議雙 buffer）
//  大小 = 解析度 × 1/10 × 2 bytes(RGB565)
// ────────────────────────────────────
#define SCREEN_W  320
#define SCREEN_H  240
#define BUF_LINES 30   // 緩衝行數，越大越快但佔 RAM

static lv_color_t disp_buf1[SCREEN_W * BUF_LINES];
static lv_color_t disp_buf2[SCREEN_W * BUF_LINES];  // 雙 buffer 效率更好

// ────────────────────────────────────
//  ① flush_cb：LVGL 叫你把像素傳出去
// ────────────────────────────────────
static void my_flush_cb(lv_display_t *disp,
                         const lv_area_t *area,
                         uint8_t *px_map)
{
    uint32_t w = area->x2 - area->x1 + 1;
    uint32_t h = area->y2 - area->y1 + 1;

    tft.startWrite();
    tft.setAddrWindow(area->x1, area->y1, w, h);
    // px_map 是 RGB565 16-bit 資料
    tft.pushColors((uint16_t *)px_map, w * h, true);
    tft.endWrite();

    // ★ 這行不能漏！告訴 LVGL 已經傳完，可以繼續
    lv_display_flush_ready(disp);
}

// ────────────────────────────────────
//  ② touch_read_cb：LVGL 問你觸控狀態
// ────────────────────────────────────
static void my_touch_read_cb(lv_indev_t *indev,
                              lv_indev_data_t *data)
{
    // 讀取 XPT2046
    // 實際上你要用對應的 XPT2046 library
    uint16_t tx, ty;
    bool pressed = tft.getTouch(&tx, &ty, 600);

    if (pressed) {
        data->point.x = tx;
        data->point.y = ty;
        data->state = LV_INDEV_STATE_PRESSED;
    } else {
        data->state = LV_INDEV_STATE_RELEASED;
    }
}

// ────────────────────────────────────
//  setup()
// ────────────────────────────────────
void setup() {
    Serial.begin(115200);

    // ─ 初始化螢幕 ─
    tft.begin();
    tft.setRotation(1);  // 橫屏
    tft.fillScreen(TFT_BLACK);

    // ─ 初始化 LVGL ─
    lv_init();

    // ─ 建立 display 物件 ─
    lv_display_t *disp = lv_display_create(SCREEN_W, SCREEN_H);
    lv_display_set_flush_cb(disp, my_flush_cb);

    // 設定雙 buffer（partial rendering 模式）
    lv_display_set_buffers(disp,
        disp_buf1, disp_buf2,
        sizeof(disp_buf1),
        LV_DISPLAY_RENDER_MODE_PARTIAL);

    // ─ 建立輸入裝置（觸控）─
    lv_indev_t *indev = lv_indev_create();
    lv_indev_set_type(indev, LV_INDEV_TYPE_POINTER);
    lv_indev_set_read_cb(indev, my_touch_read_cb);

    // ─ 建立你的 UI ─
    my_ui_init();

    Serial.println("LVGL 啟動完成！學姊表示滿意。");
}

// ────────────────────────────────────
//  loop()：越快執行越好，不要加 delay()
// ────────────────────────────────────
void loop() {
    // ★ 核心：讓 LVGL 的定時器/動畫/渲染跑起來
    //   lv_tick_inc 由 lv_conf.h 中設定為 millis() 自動提供
    uint32_t sleep_ms = lv_timer_handler();
    delay(sleep_ms);  // 用 LVGL 回傳值決定 delay 時間，最省電
}

// ────────────────────────────────────
//  建立 UI
// ────────────────────────────────────
void my_ui_init() {
    lv_obj_t *scr = lv_screen_active();

    // 設定背景色
    lv_obj_set_style_bg_color(scr, lv_color_hex(0x0d0f1a),
                               LV_PART_MAIN);

    // 建立標籤
    lv_obj_t *label = lv_label_create(scr);
    lv_label_set_text(label, "Hello from 學姊!");
    lv_obj_set_style_text_color(label, lv_color_hex(0xe879a0),
                                 LV_PART_MAIN);
    lv_obj_set_style_text_font(label, &lv_font_montserrat_20,
                                LV_PART_MAIN);
    lv_obj_align(label, LV_ALIGN_CENTER, 0, -30);

    // 建立按鈕
    lv_obj_t *btn = lv_button_create(scr);
    lv_obj_set_size(btn, 140, 45);
    lv_obj_align(btn, LV_ALIGN_CENTER, 0, 30);
    lv_obj_set_style_bg_color(btn, lv_color_hex(0xe879a0),
                               LV_PART_MAIN);
    lv_obj_set_style_radius(btn, 22, LV_PART_MAIN);
    lv_obj_add_event_cb(btn, [](lv_event_t *e) {
        LV_UNUSED(e);
        Serial.println("按鈕被按了！你真的很愛亂按...");
    }, LV_EVENT_CLICKED, NULL);

    lv_obj_t *btn_lbl = lv_label_create(btn);
    lv_label_set_text(btn_lbl, "點我啊 笨蛋");
    lv_obj_center(btn_lbl);
}

                                     uint32_t arc_color)
{

記憶體：最容易翻車的地方
ESP32 只有 320KB RAM，用一點少一點，不能亂浪費。
👩‍💻
這裡是 99% 的人卡關的地方。 認真聽。

LVGL 需要：Frame Buffer（畫面快取） + lv_mem（LVGL 自用 heap）。
Frame Buffer 越大，畫面更新越快（可以一次傳更多行），但也吃更多 RAM。

黃金比例：Frame Buffer = 螢幕寬 × 行數 × 2（RGB565）， 行數建議 1/10 解析度，大概 20~40 行，超過就是在浪費 RAM 了。

📦 ESP32 320KB RAM 典型分配（LVGL 專案）

📊 不同設定下的記憶體佔用估算

~19 KB
Frame Buffer × 2
~38 KB

~48 KB

~80 KB

~80 KB

~55 KB
⚠️ 注意：全螢幕 Buffer 是天坑！ 320×240×2 bytes = 153,600 bytes = 150 KB。ESP32 根本放不下。 請務必使用 Partial Rendering（部分渲染），只備份幾行的 buffer。
✅ ESP32-S3 + PSRAM 是最佳解 如果你的板子有 PSRAM（外部 SPI RAM），可以把 Frame Buffer 放到 PSRAM 去， 這樣就能用大 buffer 甚至全螢幕 buffer，效能大提升。 heap_caps_malloc(size, MALLOC_CAP_SPIRAM)
psram_buffer.cpp — 使用 PSRAM 開大 buffer
// 確認開啟 PSRAM：platformio.ini 加上
// board_build.arduino.partitions = default_8MB.csv
// 或在 Arduino IDE 選 PSRAM: "Enabled"

#include "esp_heap_caps.h"

// 在 PSRAM 分配全螢幕 buffer（需要 ~150KB PSRAM）
static lv_color_t *disp_buf1 = nullptr;
static lv_color_t *disp_buf2 = nullptr;

void init_display_buffers() {
    size_t buf_size = SCREEN_W * SCREEN_H * sizeof(lv_color_t);

    // 嘗試從 PSRAM 分配
    disp_buf1 = (lv_color_t *)heap_caps_malloc(buf_size, MALLOC_CAP_SPIRAM);
    disp_buf2 = (lv_color_t *)heap_caps_malloc(buf_size, MALLOC_CAP_SPIRAM);

    if (!disp_buf1 || !disp_buf2) {
        Serial.println("PSRAM 不夠，改用 SRAM partial buffer");
        // Fallback 到小 buffer
        static lv_color_t fb1[SCREEN_W * 30];
        static lv_color_t fb2[SCREEN_W * 30];
        disp_buf1 = fb1;
        disp_buf2 = fb2;
        buf_size = sizeof(fb1);
    }

    lv_display_t *disp = lv_display_create(SCREEN_W, SCREEN_H);
    lv_display_set_flush_cb(disp, my_flush_cb);
    lv_display_set_buffers(disp, disp_buf1, disp_buf2,
                           buf_size, LV_DISPLAY_RENDER_MODE_FULL);

    Serial.printf("Buffer 分配成功，每個大小: %d bytes\n", buf_size);
    Serial.printf("可用 PSRAM: %d bytes\n", esp_get_free_heap_size());
}

📜 十戒（學姊親訂，如有違者後果自負）

① 絕對不要在 loop() 裡加 delay() delay() 會卡住 lv_timer_handler()，動畫變卡頓、觸控失靈。要等待就用 lv_timer 或 millis() 非阻塞寫法。
② lv_display_flush_ready() 不能漏 flush_cb 結尾沒有呼叫這個，畫面只更新一次然後永遠卡死。我說的。
③ LVGL 不是 thread-safe 如果你用 FreeRTOS（ESP32 預設就有），所有 LVGL 操作必須在同一個 task 裡， 或者用 mutex 鎖起來。跨 task 改 UI = 隨機當機。
④ 不要在 event callback 裡直接刪除物件 在 callback 裡呼叫 lv_obj_delete() 刪掉自己，LVGL 會瞬間爆炸。 要刪請用 lv_obj_delete_async()。
⑤ lv_conf.h 設定錯解析度 = 顯示花掉 LV_HOR_RES 跟 LV_VER_RES 必須跟你的螢幕物理像素完全一致，不是「差不多」就好。
⑥ 用 lv_obj_align() 而不是 lv_obj_set_pos() set_pos 是絕對座標，換螢幕分辨率全部爆掉。align 是相對對齊，才是正確姿勢。
⑦ 字型問題：中文需要額外處理 LVGL 內建字型沒有中文。要顯示中文，需要用 LVGL Font Converter 把 TTF 轉成 C 陣列， 或者用 LittleFS 載入。不要以為裝完就有了。
⑧ SPI 速度太慢 = 畫面撕裂 TFT_eSPI 的 SPI_FREQUENCY 建議 40MHz 以上。太慢的話每次更新畫面都看得到掃描線。 ESP32 SPI 最高支援 80MHz。
⑨ 記得設 LV_COLOR_DEPTH = 16 TFT 螢幕幾乎都是 RGB565（16-bit），如果設成 32 bit，顏色會整個跑掉而且多吃一倍記憶體。
⑩ 用官方模擬器先測試 LVGL 有提供 PC 模擬器（就是你那個 lv_port_pc_vscode！）。 在電腦上測邏輯、測動畫，確定沒問題再燒到 MCU，省你九成 debug 時間。
